### Especificação de Caso de Uso: Projeto Legixtech

#### UC01 – Acessar Resumo Simples e Impacto

* **Ator Principal:** Estudante Universitário (Lucas Mendes).
* **Objetivo:** Entender o impacto prático de uma lei lendo no máximo 3 parágrafos claros, sem termos jurídicos.
* **Pré-condição:** Usuário na tela de detalhes de um projeto de lei.
* **Fluxo Principal:**
    1. O usuário abre um projeto de lei extenso. 
    2. O sistema renderiza, por padrão, a aba "Entenda a Lei", escondendo o texto integral por trás de um menu. 
    3. O sistema exibe um bloco chamado "O Resumo", estruturado em exatamente três parágrafos curtos.
    4. O usuário lê o resumo e rola a tela para o próximo bloco, chamado "Quem é afetado?". 
    5. O sistema lista em tópicos práticos (bullet points) os grupos sociais ou setores econômicos impactados pela medida. 
    6. Para citar o projeto na faculdade, o usuário toca no link inferior rotulado "Acessar Texto Oficial". 
    7. O sistema redireciona o usuário (ou abre um in-app browser) para o portal do Senado/Câmara.
* **Fluxo Alternativo (Texto Oficial indisponível):** 
    1. O usuário clica em "Acessar Texto Oficial", mas a API do governo está fora do ar. 
    2. O sistema exibe uma mensagem: "O documento original está indisponível na fonte oficial no momento. Tente novamente mais tarde."
* **Pós-condição:** O usuário adquire compreensão técnica e prática do projeto de forma ágil.
